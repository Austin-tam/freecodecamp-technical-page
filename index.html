Introduction

Python is an interpreted, object-oriented programming language. Created by Guido van Rossum and released in 1991, Python is one of the most popular choices for backend development, being employed by websites such as Instagram, Uber, Dropbox, and many more.

The main feature of python is that it is made to be beginner friendly and easy to learn. Other features of the language include:

  • Easy-to-read: The simplistic syntax makes it easy and quick to read the code
  • Easy-to-maintain: Since the code is easy to read, it is therefore easy to maintain
  • Portable: Python can run on many different hardware platforms and shares the same interface across all of these platforms
  • Large Standard Library: Python has a large standard library which provides a set of many useful built-in modules and functions for a variety of uses

Syntax

The definition of syntax is "the arrangement of words and phrases to create well-formed sentences in a language". We can apply this definition to programming languages as well. Syntax, in the context of a programming language, refers to the structure of the language, and the arrangement of words and phrases. Although Python was originally developed as a teaching language, its easy-to-use nature and clean syntax has appealed to beginners and experts as well. Python uses English words in places where other languages would use punctuation.

For example, take the following Java code:

int[] values = new int[100];
for (int i=0; i<values.length; i++) { }

Now compare it to the following Python code:

values = [0] * 10
for i in range(0, len(values), 1):

As you can tell, the for loop declaration in Python is "easier" to read since it uses more English words and one can instinctively understand what is happening. Whereas with the Java code, one would have to look through each section of the for loop to understand each part.

Another example is Python's use of Not in place of ! that many other languages use. With a larger code base, finding and reading the keyword Not is easier than ! which makes it harder to miss and harder to make errors.

Variables

In programming, variables are used to store data values. Python is a strongly and dynamically typed language. Strong meaning that the type of a value does not change in unexpected ways, and dynamic meaning that Python checks the types of values at runtime rather than at compile time. Because of this, variables in Python code are very easy to create and initialize.


For example, a simple variable declaration can look like this:

x = 0

This tells Python to create an integer object with a value of 0 and have the variable "x" point to that object. Similarly, when you declare strings like this:

y = "text"

Python creates a string object with the value of "text" and has the variable "y" point to that object.

Python has very little restrictions in terms of variable names. They can be any length and can consist of any combination of uppercase and lowercase letters (A-Z, a-z), digits (0-9), and the underscore character (_). The main restriction is that the first character of a variable name cannot be a digit. The only other restriction is that variable names in Python cannot be the same as any of the keywords that Python has reserved for special functions. These keywords include:

  • False
  • True
  • if
  • None
  • not
  • and
  • def

Control Flow

Control flow, in programming, refers to the order in which statements, instructions, and/or function calls are executed. Control flow is important to know as most of the time there will be situations where the program will need to do different things based on different states. In other cases, the program may need to break the flow of the program to execute code multiple times. For cases like these, programmers use control flow statements to handle the flow of the program.

These control flow statements include:

  • The if statement
  • The For loop
  • The while loop
  • The break and continue statements

There are many other control flow statements but these are the ones we will be focusing on in this section.

The if statement is one of the most common control flow statements frequently used in programming. In Python, the full if statement looks like this:

if . . elif . . else

With this statement, the program will execute code if a logical condition is true. If false, the program will check the "elif", short for "else if", conditional statements and if true, will execute that code. If none of the conditional statements return true, it will execute the code in the "else" block. The conditional statements must evaluate to be true or false for the if statement to work properly, and there can be any number of elif statements. Python also does not have a switch . . case statement like many other languages use.

Another commonly used control flow statement is the for loop. This statement is used to loop through a block of code, as long as the condition of the for loop is true. To create a for loop in Python, you would do something similar to this:

for . . in . . else

the for part of the statement starts the loop. Between the for and the in section is a variable that will be used to iterate through a range. This range is defined after the in portion of the loop. For example if the range is range(0,5) then the loop will iterate through the numbers 0, 1, 2, 3, and 4, and execute the block of code for each of those numbers. The else portion of the for loop is optional. If the program breaks out of the loop or the loop is finished, the program will check for any code in an else block and execute it if it is there.


The next control flow statement is the while loop. Although this is a loop just like the for loop, the functionality between the two are different. The syntax for the while loop is as follows:

while . . else

Where the while keyword starts the loop, and there is a condition after the while keyword. The while statement will continue to loop through the block of code, so long as the condition remains true. The else part of the loop is optional, same as the for loop, and will execute when the conditional of the while loop is false.

The last control flow statements in this documentation are the break and continue keywords. The break keyword is very useful because it allows the program to prematurely break out of a loop if it needs to. The break keyword is used by itself on its own line and does not require any other parameters. When it is used, the program will break out of the current loop that it is in, and continue to execute the rest of the code down the program.

The continue keyword is similar to the break keyword in that they both control the flow of loops. When you use the continue keyword, it tells the program to continue to the next step of the loop. For example, take the following code:

for x in range(0,5,1):
  if (x % 2 == 0):
   print("found an even number")
   continue
  print("found an odd number")

You will see that we have a for loop from 0 to 5, with a step of 1. Inside the for loop, we have an if statement with a conditional statement that will be true when the remainder of x/2 is equal to 0; i.e if the number is even. When x is an even number, the program will print out the string "found an even number" and then the continue keyword will force the loop to step to the next iteration of the loop.

Functions

Functions in Python are a very powerful tool that assist every aspect of a program including readability, scalability, cleanliness of code, and organization. Functions should not be confused with methods. The main difference between the two are that methods are functions that are a part of a class, while a function does not have this restriction.

Creating functions in Python are very easy. The first line of the function looks similar to this:

def function_name(parameter_1, parameter_2):

Where def is the keyword that starts the function, followed by the name of the function. The function may also have optional parameters that the program can pass into. After definition the function, the tab-indented code will be executed every time the main program calls the function. Returning a value at the end of a function is optional and the function will work whether there is a return line or not.

There are very few restrictions on the names of functions, which are similar to variable names. Namely, they can contain both uppercase and lowercase characters, numbers, and underscores. Also similarly to variables, function names cannot be the same as reserved keywords and reserved functions. They also cannot start with a number. Other than these restrictions, any combination of characters, numbers, and underscores are allowed.

The next topic to comment on about function names is naming convention. Although any combination of characters is allowed, there are generally accepted rules that programmers use to keep function names universal. Some of these rules include:

  • Lowercase, with words separated by underscores to improve readability
  • mixedCase is allowed only if it is the style that is being used, to stay consistent
  • Always use self for the first argument to instance methods
  • Always use cls for the first argument to class methods
  • If a function name clashes with a reserved keyword or function, it is generally accepted convention to add an underscore to the end of the function name, rather than abbreviating or altering the spelling

Libraries

Much like other languages, Python has a large, built-in, standard library that is available for use. To use this library, you use Python's keyword import. The way import works in Python is similar to Java's import, in that the interpreter looks through the standard library for the module that is being imported. If not found, it searches through the directories listed in sys.path. One can create their own custom modules and save it in sys.path to be imported and used in later programs.

In Python's standard library, there are a resourceful amount of modules that help fulfill a wide variety of programming needs. Some of the popular modules include:

  • math - containing many math functions such as ceiling, floor, square root, exponent, logarithm, etc
  • csv - used or CSV file reading and writing
  • tkinter - Python's interface to the Tk GUI toolkit
  • datetime - basic date and time types
  • random - used to generate pseudo-random numbers
